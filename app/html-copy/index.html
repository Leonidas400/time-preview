Sobre o Projeto

Este projeto tem como objetivo oferecer uma aplicação web simples e eficiente para consultar a previsão do tempo de cidades brasileiras em tempo real. A interface é intuitiva, permitindo ao usuário buscar uma cidade e visualizar informações como temperatura atual, sensação térmica, umidade, condição climática, velocidade do vento e as temperaturas mínima e máxima. Todas as informações são organizadas de forma visualmente agradável para facilitar a compreensão rápida do clima local.

O sistema foi pensado para ser leve, acessível e fácil de executar em diferentes ambientes. Com o uso de tecnologias modernas como contêineres Docker, a aplicação pode ser executada com facilidade em qualquer máquina que suporte essa tecnologia, sem necessidade de configurar dependências manualmente. Essa abordagem garante portabilidade, reprodutibilidade e praticidade para estudantes, desenvolvedores e qualquer pessoa que deseje uma forma rápida e confiável de consultar o clima.

Como Utilizar

Para utilizar a aplicação, o usuário deve digitar o nome da cidade desejada no campo de busca localizado na parte superior da interface e clicar no botão "Pesquisar". Imediatamente, os dados climáticos da cidade informada serão apresentados, acompanhados de um mapa da região. As informações são exibidas em blocos organizados, com ícones ilustrativos que ajudam na leitura e compreensão dos dados.

Como Funciona

A aplicação se conecta à API pública do OpenWeatherMap, que fornece dados meteorológicos atualizados. Ao receber o nome da cidade informado pelo usuário, o backend realiza uma requisição para a API, processa os dados e os envia ao front-end, onde são exibidos com o uso de HTML e CSS. Um mapa da cidade, com base nas coordenadas geográficas fornecidas pela própria API, também é incorporado à interface para complementar a experiência visual e geográfica do usuário.

Como Foi Desenvolvido

A aplicação foi desenvolvida utilizando a linguagem Python com o framework Flask para estruturar o backend. No entanto, o principal foco técnico do projeto está na sua containerização com Docker. Toda a aplicação foi empacotada em um container Docker, permitindo que ela seja executada de maneira padronizada e independente do sistema operacional do host.

A imagem Docker criada inclui todas as dependências da aplicação, como o Flask, o Requests e o interpretador Python 3.12. Com isso, o processo de execução do projeto torna-se extremamente simples e confiável: basta construir a imagem e executá-la com um único comando. O uso de Dockerfiles permite que qualquer desenvolvedor ou usuário replique o ambiente da aplicação sem complicações, eliminando problemas com versões de bibliotecas, conflitos de ambiente ou configurações específicas da máquina.

Além disso, o mapeamento de portas entre o container e o host garante que a aplicação esteja acessível pela interface web localmente. Esse processo mostra na prática como o Docker pode ser utilizado para facilitar a entrega, teste e implantação de aplicações em ambientes reais ou simulados.
